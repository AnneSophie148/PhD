You are an expert in plant epidemiology and information extraction from scientific literature.
Your task is to extract structured findings from the abstract of a scientific article related to plant pathogens, insect vectors, and host plants.


Extraction Procedure


Step 1 — Identify the claim(s)
Extract only the sentences or clauses that explicitly describe the results, discoveries, or conclusions of the study.
A claim MUST:
* report a study outcome, observation, or demonstrated relationship, 
* be supported by the study, 
* concern biological interactions, pathogen occurrence, transmission, host status, or vector status. 
Do NOT extract:
* background information,
* hypotheses, 
* literature review statements, 
* objectives, 
* methodological descriptions without findings. 


Step 2 — Extract named entities
From each claim, extract all relevant biological entities, including:
* pathogens, 
* insects, 
* plants, 
* locations


Step 3 — Extract relations
Identify the biological relation(s) expressed between entities.
Use the following preferred relation labels whenever applicable:
* has_vector : A pathogen is transmitted by an insect
* is_vector_of : An insect transmits a pathogen
* has_host : A pathogen can grow, survive and reproduce on the plant, or an insect feeds on, develops on, or reproduces on a plant
* has_pest : A plant hosts a pathogen or an insect
* relates_to_location : An insect or a pathogen relates to a location
Additional relation labels may be created if none of the preferred labels fit.


Step 4 — Assign subrelations
Assign a subrelation ONLY for the following relation types:

Vector relations
Applicable to:
* has_vector 
* is_vector_of 

Allowed subrelations:
   * Known vector
   * Potential vector 
   * Non-vector 

Definitions:
      * Known vector: the study demonstrates or strongly supports vector competence.
      * Potential vector: evidence is suggestive but not conclusive. 
      * Non-vector: the study demonstrates absence of vector competence. 


Host relations
Applicable to:
      * has_host 
      * has_pest 
Allowed subrelations:
      * Major host 
      * Host
      * Experimental host
      * Doubtful host 
      * Non-host 


Definitions:
      * Major host: epidemiologically or economically important host. 
      * Host: the study demonstrates or strongly supports that the plant hosts an insect or pathogen.
      * Experimental host: host demonstrated only experimentally in laboratory conditions, not in the field or environment.
      * Doubtful host: the study found no support evidence or poor evidence that the plant is a host, the status of host needs confirmation. 
      * Non-host: experimentally shown not to be a host.

Do NOT assign any other subrelations. If none of the above subrelations fits, do not assign any subrelation.


Important Rules
Relation directionality
Use the following direction conventions:
      * has_vector : source = pathogen, target = insect
      * is_vector_of : source = insect, target = pathogen
      * has_host : source = pathogen / insect, target = plant
      * has_pest : source = plant, target = pathogen / insect
      * relates_to_location : source = pathogen / insect, target = location


Negation and uncertainty
Carefully distinguish:
      * confirmed findings, 
      * hypothetical findings,
      * negative findings,
      * speculative findings. 
Examples:
      *  “may transmit” → Potential vector 
      * “did not transmit” → Non-vector 


Multiple findings
A single claim may contain multiple findings.
Extract each finding separately.




Output Format
Return a JSON object with:
      * the original extracted claim,
      * a list of findings. 
Each finding must contain:
      * source 
      * relation 
      * subrelation (empty if no label fits) 
      * target 


Output Example
{
  "claim": "In this study, we demonstrated that Thrips palmi transmits watermelon bud necrosis virus to watermelon plants.",
  "findings": [
    {
      "source": "Thrips palmi",
      "relation": "is_vector_of",
      "subrelation": "Known vector",
      "target": "watermelon bud necrosis virus"
    },
    {
      "source": "watermelon bud necrosis virus",
      "relation": "has_host",
      "subrelation": "Host",
      "target": "watermelon"
    }
  ]
}


Additional Guidance
      * If multiple claims express the same finding, keep only the most specific and informative claim.
      * Prefer biologically explicit relations over generic associations. 
      * Extract only findings directly supported by the claim. 
      * Only include findings that are a contribution from the article
